# セキュリティ

[English](../SECURITY.md) | 日本語

## 監査サマリー

**2026-05-09** に、`novelai-sdk` 依存関係およびこのリポジトリのリレーサーバー（`src/python/`）を対象とした内部セキュリティ監査を実施しました。

**結果: セキュリティ上の問題は見つかりませんでした。**

---

## ネットワーク通信

SDK は以下の公式 NovelAI エンドポイントとのみ HTTPS で通信します。

| ホスト | 用途 |
|---|---|
| `https://image.novelai.net` | 画像生成 |
| `https://text.novelai.net` | テキスト生成 |
| `https://api.novelai.net` | 汎用 API |

サードパーティへのデータ送信は一切行いません。3つのベース URL はいずれも環境変数（`NOVELAI_IMAGE_BASE`、`NOVELAI_TEXT_BASE`、`NOVELAI_API_BASE`）で変更可能で、デフォルトは公式 NovelAI ドメインです。

### リレーサーバーのトポロジー

```
ブラウザ
  ↓ HTTP（localhost:8000、ローカルのみ）
リレーサーバー（src/python/）
  ↓ HTTPS
image.novelai.net  ← 公式 NovelAI サーバー
```

リレーサーバーはローカルマシン上のみで動作し、外部へデータを公開しません。

---

## 認証

- API トークンは標準的な `Authorization: Bearer {token}` ヘッダーとして渡されます。
- トークンはディスクへの書き込み、ログ出力、リクエストボディや URL への含め込みは一切行われません。
- `NOVELAI_API_KEY` / `NOVELAI_API_TOKEN` 環境変数（または `.env` ファイル）から読み込まれ、メモリ上のみで保持されます。

## コード検査

`novelai-sdk` において以下の危険なパターンを検索しましたが、**いずれも見つかりませんでした。**

- 動的コード実行: `eval()`、`exec()`、`compile()`、`__import__()`
- シェルアクセス: `subprocess`、`os.system()`、`popen()`
- 難読化コードや不審なインポート

ファイル I/O は、ユーザーが指定した画像ファイルの読み込みと、CLI 経由でユーザーが指定した出力ディレクトリへの生成画像の書き込みのみに限定されています。

## 認証情報の取り扱い

- API トークンは `.env` にのみ保存してください。ソースファイルにハードコードしないでください。
- `.env` は `.gitignore` に記載されており、リポジトリにコミットされません。
- `.env` やその内容を公開しないでください。
